# 사용자 패턴 분석 계획서

## 1) 목적
- 사용자 행동 데이터를 기반으로 **핵심 사용 패턴**을 식별한다.
- 리텐션, 전환, 이탈 구간을 정량적으로 파악해 제품 개선 우선순위를 도출한다.
- 분석 결과를 기능 개선/운영 정책/마케팅 실험에 연결 가능한 형태로 문서화한다.

## 2) 분석 질문(핵심 가설)
1. 신규 사용자는 첫 24시간 내 어떤 행동을 할 때 다음 날 재방문 확률이 높아지는가?
2. 고활성 사용자와 저활성 사용자의 이벤트 시퀀스는 어떻게 다른가?
3. 특정 기능(예: 미션 완료, 연속 접속, 공유 등) 사용이 결제/구독/잔존율에 어떤 영향을 주는가?
4. 이탈 직전 공통 패턴(세션 길이 감소, 특정 화면 반복 진입, 에러 발생 등)이 존재하는가?

## 3) 분석 범위
- 기간: 최근 90일(주 단위 비교 포함)
- 대상: 앱 설치 후 30일 이내 사용자 + 장기 사용자(30일 초과) 분리 분석
- 플랫폼: Android 우선, iOS는 데이터 수집 체계 동등성 확인 후 확장
- 제외: 내부 테스트 계정, 비정상 트래픽(봇/자동화/중복 설치)

## 4) 데이터 수집 설계

### 4.1 이벤트 택소노미(초안)
- `app_open` : 앱 실행
- `session_start`, `session_end` : 세션 시작/종료
- `view_screen` : 화면 진입 (`screen_name` 필수)
- `mission_start`, `mission_complete` : 미션 시작/완료
- `streak_check`, `streak_break` : 연속 접속 확인/중단
- `share_click`, `share_complete` : 공유 시도/완료
- `subscription_view`, `subscription_start`, `subscription_success` : 구독 퍼널
- `error_client`, `error_api` : 오류 이벤트

### 4.2 필수 속성(공통)
- `user_id`(해시), `session_id`, `event_time(UTC)`
- `platform`, `app_version`, `locale`, `country`
- `acquisition_channel`(organic/ad/campaign)
- `screen_name`, `feature_name`(해당 시)

### 4.3 데이터 품질 기준
- 누락률: 핵심 이벤트 누락 < 2%
- 이벤트 중복률: < 1%
- 이벤트 지연 적재: D+1 오전 9시 이전 99% 반영
- 스키마 변경 시: 변경 이력/버전 명시 및 하위 호환 점검

## 5) 지표 정의

### 5.1 활성/잔존
- DAU/WAU/MAU
- D1/D7/D30 Retention
- Stickiness = DAU / MAU

### 5.2 퍼널
- 온보딩 완료율
- 핵심 기능 도달율
- 구독 전환율(View → Start → Success)

### 5.3 행동 패턴
- 평균 세션 수/일, 평균 세션 길이
- 사용자별 핵심 이벤트 시퀀스 Top N
- 기능 재사용 간격(재방문 주기)

### 5.4 안정성
- Crash-free 사용자 비율
- 오류 발생 사용자 비율 및 오류 후 이탈률

## 6) 분석 방법론
1. **코호트 분석**: 설치 주차/유입 채널/버전별 잔존 비교
2. **시퀀스 분석**: 전환 사용자 vs 비전환 사용자 행동 경로 비교
3. **세그먼트 분석**: 고활성/저활성/휴면 직전 사용자 분류
4. **상관 탐색**: 특정 행동과 리텐션/전환 간 상관관계 검토
5. **실험 설계 연계**: 분석 결과 기반 A/B 테스트 가설 생성

## 7) 운영 절차

### 7.1 주간 루틴
- 월: 전주 핵심 지표 스냅샷 업데이트
- 화: 이상 징후(급락/급증) 원인 분석
- 수: 사용자 세그먼트별 패턴 분석
- 목: 개선 과제/실험안 정리
- 금: 결과 공유 및 다음 주 액션 확정

### 7.2 산출물
- 주간 패턴 분석 리포트(요약 + 차트 + 액션 아이템)
- 월간 코호트/퍼널 심화 분석 보고서
- 이벤트 스키마 변경 로그

## 8) 구현 일정(초안)
- 1주차: 이벤트 정의 확정, 로깅 점검, 데이터 품질 대시보드 준비
- 2주차: 기초 지표(활성/잔존/퍼널) 자동 리포트 구축
- 3주차: 세그먼트/시퀀스 분석 및 이탈 패턴 탐색
- 4주차: 개선안 도출, 실험 우선순위 수립, 다음 분기 로드맵 반영

## 9) 리스크 및 대응
- 리스크: 이벤트 누락/중복 → 대응: 수집 검증 쿼리 자동화, 릴리즈 체크리스트 반영
- 리스크: 지표 정의 혼선 → 대응: 메트릭 정의서 단일화 및 용어 사전 관리
- 리스크: 개인정보 이슈 → 대응: 식별자 해시 처리, 최소 수집 원칙, 접근 권한 통제

## 10) 의사결정 원칙
- 단일 지표가 아닌 **잔존 + 전환 + 안정성**을 함께 본다.
- 단기 상승보다 재현 가능한 패턴과 장기 유지 효과를 우선한다.
- 분석 결과는 반드시 “실행 가능한 액션”으로 변환해 스프린트 백로그에 반영한다.
