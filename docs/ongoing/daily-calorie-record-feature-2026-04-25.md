# 일일 칼로리 기록 기능 추가 상세 정리

- 작성일: 2026-04-25

- 원문 제목: Add daily calorie record feature
- 문서 성격: 기능 요구사항/설계 초안 (ongoing)

## 1) 기능 목표
사용자가 **하루 단위로 섭취 칼로리와 소모 칼로리**를 기록하고, 순칼로리(섭취-소모) 및 목표 대비 달성률을 확인할 수 있도록 한다.  
핵심 목표는 다음과 같다.

1. 사용자가 식단/운동 기록을 빠르게 입력할 수 있어야 한다.
2. 하루 합계/순칼로리를 즉시 계산해서 보여줘야 한다.
3. 최근 7일/30일 추세를 통해 습관 개선 인사이트를 제공해야 한다.
4. 기존 퀘스트/루틴 기능과 연결 가능한 확장성을 가져야 한다.

---

## 2) 사용자 시나리오

### 시나리오 A: 식사 후 섭취 칼로리 기록
1. 사용자가 "칼로리 기록" 화면 진입
2. "섭취 추가" 버튼 탭
3. 음식명(선택), 칼로리(kcal), 메모(선택), 시간 입력
4. 저장 즉시 해당 날짜 섭취 총합 업데이트

### 시나리오 B: 운동 후 소모 칼로리 기록
1. 사용자가 "소모 추가" 버튼 탭
2. 운동명(선택), 소모 칼로리(kcal), 시간, 메모 입력
3. 저장 즉시 해당 날짜 소모 총합 및 순칼로리 갱신

### 시나리오 C: 일간/주간 추세 확인
1. 사용자가 오늘 카드에서 "최근 7일" 탭 전환
2. 날짜별 섭취/소모/순칼로리 막대 또는 라인 차트 확인
3. 목표 초과/미달 상태를 색상 배지로 확인

## 3) 기능 요구사항 (Functional Requirements)

### FR-01. 일일 칼로리 요약
- 날짜별로 다음 값을 계산/표시한다.
  - 섭취 총합(`intakeTotalKcal`)
  - 소모 총합(`burnTotalKcal`)
  - 순칼로리(`netKcal = intakeTotalKcal - burnTotalKcal`)
  - 목표 대비 달성률(`progressPercent`)

### FR-02. 기록 CRUD
- 섭취/소모 항목 각각 생성(Create), 수정(Update), 삭제(Delete) 가능
- 항목 필수값: 타입(섭취/소모), 칼로리, 기록 시각
- 옵션값: 제목(음식/운동명), 메모, 태그

### FR-03. 날짜 단위 조회
- 기본 조회 범위: 오늘
- 추가 조회: 특정일, 최근 7일, 최근 30일
- 타임존 기준은 사용자 로컬 타임존으로 계산

### FR-04. 목표 설정
- 사용자별 일일 목표 칼로리 설정 가능
- 목표 미설정 시 기본값(예: 2000kcal) 또는 "미설정" 상태로 처리

### FR-05. 상태 피드백
- 목표 초과/적정/미달에 따라 색상 상태 표시
- 저장/수정/삭제 시 토스트 피드백 제공

## 4) 비기능 요구사항 (Non-Functional Requirements)
- 입력 반응성: 저장 액션 후 1초 이내 UI 반영
- 데이터 정합성: 중복 저장/음수 값 저장 방지
- 확장성: 향후 음식 DB 연동 및 자동 칼로리 추정 기능 추가 가능 구조
- 접근성: 숫자 입력 필드 라벨, 버튼 접근성 텍스트 제공

## 5) 데이터 모델 초안

### 컬렉션 구조 예시 (Firestore)
`users/{uid}/calorieLogs/{logId}`

```json
{
  "type": "intake",          // "intake" | "burn"
  "kcal": 450,
  "title": "점심 - 닭가슴살 샐러드",
  "memo": "드레싱 반만 사용",
  "loggedAt": "2026-04-25T12:30:00+09:00",
  "localDate": "2026-04-25", // 집계 키
  "createdAt": "serverTimestamp",
  "updatedAt": "serverTimestamp"
}
```

### 사용자 설정 예시
`users/{uid}/settings/nutrition`

```json
{
  "dailyCalorieGoal": 1800,
  "updatedAt": "serverTimestamp"
}
```

## 6) 화면/UX 초안

### A. 오늘 요약 카드
- 섭취: `1,650 kcal`
- 소모: `420 kcal`
- 순칼로리: `1,230 kcal`
- 목표 대비: `68%`

### B. 기록 리스트
- 시간순 정렬(최신 우선)
- 항목 좌측 배지: 섭취(+) / 소모(-)
- 항목 스와이프 또는 메뉴: 수정/삭제

### C. 입력 모달
- 필드: 타입, kcal, 제목, 시간, 메모
- 유효성 오류 메시지 즉시 표기

---

## 7) 유효성/에러 처리 규칙
1. kcal은 1~5000 범위 정수만 허용
2. 미래 시각 기록은 허용하되 기본값은 현재 시각
3. 수정/삭제 시 대상 문서 없으면 재조회 후 경고 토스트 표시
4. 네트워크 오류 시 오프라인 큐 또는 재시도 버튼 제공

## 8) 구현 단계 제안

### Phase 1 (MVP)
- 기록 생성/조회/삭제
- 오늘 요약(섭취/소모/순칼로리)

- 음식/운동 템플릿
