## 1. 저작권 등록의 필요성

### 1-1. 왜 등록해야 하는가

- **법적 추정력 확보**: 저작권은 창작과 동시에 발생하나, 등록 시 저작자·창작일·공표일 등이 법적으로 추정됨 (저작권법 제53조)
- **분쟁 시 입증 부담 경감**: 침해소송에서 등록증이 핵심 증거 역할 수행
- **손해배상 청구 강화**: 등록된 저작물은 법정손해배상 청구 가능 (저작권법 제125조의2)
- **제3자 대항력**: 저작재산권 양도·질권설정 등의 등록은 제3자 대항요건

### 1-2. 미등록 시 리스크

| 리스크 | 영향 |
|--------|------|
| 타인의 선등록 | 저작자 지위를 입증해야 하는 부담 발생 |
| 침해 소송 시 증거 부족 | 창작 시점·범위 입증 곤란 |
| 투자/M&A 실사 | IP 자산 가치 증명 어려움 |
| 해외 진출 시 보호 공백 | 국가별 등록 없이는 현지 보호 미흡 |

## 3. 한국 저작권 등록 절차

### 3-1. 등록 기관

- **한국저작권위원회** (Korea Copyright Commission)
- 온라인 등록 시스템: https://www.cros.or.kr (저작권등록시스템)

### 3-2. 등록 유형

| 유형 | 설명 | 적용 |
|------|------|------|
| **저작자 등록** | 저작자의 실명/이명 등록 | 개인 또는 법인 명의 |
| **창작연월일 등록** | 창작 시점 공적 기록 | 소스코드 최초 커밋 시점 기준 |
| **공표연월일 등록** | 최초 공표(출시) 시점 기록 | 플레이스토어 최초 배포일 |
| **권리변동 등록** | 양도·질권 등 권리변동 등록 | 향후 계약 시 활용 |

### 3-3. 등록 절차 (온라인)

```
1단계: 저작권등록시스템(CROS) 회원가입 및 로그인
    ↓
2단계: 등록 신청서 작성
    - 저작물 유형 선택: "컴퓨터프로그램저작물"
    - 저작물 명칭: "LEVEL UP: REBOOT 모바일 애플리케이션"
    - 저작자 정보: BRAVECAT STUDIOS (법인) 또는 대표자 실명
    - 창작연월일: 최초 커밋일 또는 기획 착수일
    - 공표연월일: 플레이스토어 최초 출시일
    ↓
3단계: 저작물 사본(복제물) 제출
    - 소스코드: 전체 코드를 ZIP 압축하여 업로드
    - 실행 화면: 주요 스크린샷 5~10장
    - 프로그램 설명서: 기능 명세, 아키텍처 개요
    ↓
4단계: 수수료 납부
    - 프로그램 저작물 등록: 건당 약 24,000원 (2026년 기준 확인 필요)
    ↓
5단계: 심사 및 등록증 발급
    - 처리 기간: 약 7~14 영업일
    - 등록증 수령 (전자문서 또는 우편)
```

### 3-4. 제출 서류 체크리스트

- [ ] 저작권 등록 신청서 (온라인 작성)
- [ ] 저작자 신원 확인 서류 (사업자등록증 또는 신분증 사본)
- [ ] 소스코드 사본 (ZIP 파일, 오픈소스/제3자 코드 분리)
- [ ] 프로그램 설명서 (기능·구조·실행환경 설명)
- [ ] 주요 화면 스크린샷
- [ ] 위임장 (대리 신청 시)

## 4. 프로그램 저작물 특별 등록 (한국SW저작권협회)

### 4-1. 개요

컴퓨터프로그램저작물은 한국저작권위원회 외에 **한국소프트웨어저작권협회(SPC)** 를 통해서도 등록 가능하다.

### 4-2. 소스코드 제출 시 영업비밀 보호

| 방식 | 설명 |
|------|------|
| **전체 공개** | 소스코드 전문 제출 |
| **일부 공개** | 핵심 알고리즘 부분만 마스킹 후 제출 |
| **비공개 봉인** | 소스코드를 봉인 제출, 분쟁 시에만 개봉 |

> **권장**: 앱의 핵심 로직(레벨업 알고리즘, 스트릭 시스템 등)은 **일부 공개** 또는 **비공개 봉인** 방식 활용

3. 소스코드 제출 (첫 25페이지 + 마지막 25페이지, 또는 전문)

## 10. 결론 및 권고사항

### 핵심 권고

1. **즉시 등록 권장**: 개발 중인 현 시점에서 프로그램 저작물 등록을 진행하여, 창작일에 대한 법적 추정력을 조기에 확보한다.
2. **비공개 봉인 방식 활용**: 핵심 알고리즘(레벨업·스트릭·보상 시스템)의 영업비밀을 보호하면서 등록한다.
3. **Git 이력 보전**: 커밋 히스토리는 창작 과정을 시간순으로 증명하는 강력한 보조 증거이므로, 원격 저장소 백업을 유지한다.
4. **단계별 접근**: 국내 등록 → 출시 시점 보완 등록 → 해외 등록 순으로 비용 효율적으로 진행한다.
5. **정기 검토**: 메이저 업데이트 시 등록 내용 갱신 여부를 검토하고, 유사 앱에 대한 침해 모니터링을 분기별로 실시한다.

> 본 보고서는 일반적인 가이드이며, 구체적인 등록 절차 및 법적 판단은 지식재산권 전문 변리사 또는 변호사의 자문을 받아 진행할 것을 권고한다.
