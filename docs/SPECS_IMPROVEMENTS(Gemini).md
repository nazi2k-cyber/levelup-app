# LEVEL UP: REBOOT — 기술적 개선 방안 (Technical Improvements)

본 문서는 `SPECS.md` 분석을 바탕으로 **LEVEL UP: REBOOT** 프로젝트의 성능, 보안, 사용자 경험을 극대화하기 위한 구체적인 기술적 개선안을 제안합니다.

---

## 1. 이미지 처리 및 전송 최적화 (Image & Media)

현재 정의된 이미지 규격(1:1, 4:5, 9:16)을 유지하면서도, 서비스의 로딩 속도와 데이터 비용을 최적화합니다.

### 1.1. 차세대 포맷 및 자동 리사이징
*   **제안:** Cloud Functions와 `sharp` 라이브러리를 연동하여 업로드 즉시 `WebP` 또는 `AVIF` 포맷으로 변환.
*   **상세:** 원본 저장 외에 용도별(Thumbnail: 200px, Medium: 800px)로 자동 리사이징된 이미지를 생성하여 클라이언트에 제공.
*   **효과:** 이미지 용량 최대 70% 절감, 모바일 환경 로딩 속도 개선.

### 1.2. 클라이언트 측 사전 편집 (Client-side Cropping)
*   **제안:** `react-easy-crop` 등 라이브러리 도입.
*   **상세:** 사용자가 업로드 전 브라우저에서 직접 규격에 맞게 이미지를 자르도록 유도.
*   **효과:** 서버 연산 부하 감소, 의도치 않은 이미지 잘림 방지, UI 일관성 확보.

### 1.3. LQIP (Low Quality Image Placeholder) 도입
*   **제안:** 이미지 로드 전 10~20px 수준의 초저해상도 이미지를 블러 처리하여 우선 노출.
*   **효과:** 사용자 시각적 피드백 즉시 제공, 체감 대기 시간 단축.

---

## 2. 데이터베이스(Firestore) 설계 고도화

SNS 특유의 높은 읽기(Read) 빈도에 최적화된 설계를 적용합니다.

### 2.1. 데이터 비정규화 (Denormalization)
*   **제안:** `posts` 문서 내에 작성자의 `displayName`과 `photoURL`을 중복 저장.
*   **이유:** 포스트 목록 조회 시 `users` 컬렉션 조인(Join) 불필요, 읽기 비용 절감 및 쿼리 속도 향상.
*   **관리:** 프로필 변경 시 Cloud Functions를 통해 관련 포스트의 정보를 일괄 업데이트하는 로직 구현.

### 2.2. 분산 카운터 (Distributed Counters)
*   **제안:** '좋아요'나 '조회수' 급증 게시물을 위한 카운터 분산 관리.
*   **이유:** 단일 문서의 초당 쓰기 제한(1회)으로 인한 경합(Contention) 에러 방지.

### 2.3. 복합 인덱스(Composite Index) 최적화
*   **제안:** `status == 'published'`와 `createdAt DESC` 조합 등 주요 쿼리에 대한 인덱스 사전 설계.
*   **효과:** 대량 데이터 환경에서도 무한 스크롤 성능 보장.

---

## 3. 보안 및 아키텍처 강화 (Security & DevOps)

Firebase 환경의 보안 허점을 차단하고 운영 안정성을 확보합니다.

### 3.1. Firestore Security Rules 정교화
*   **제안:** 필드 레벨 유효성 검사(Validation) 강화.
*   **상세:** 단순히 인증 여부만 체크하는 것이 아니라, 필드 타입(string, int), 문자열 길이, 필수 필드 포함 여부를 규칙에서 강제.
*   **효과:** 악의적인 클라이언트의 데이터 조작 시도 원천 차단.

### 3.2. Cloud Functions 멱등성(Idempotency) 확보
*   **제안:** 알림 발송, 포인트 지급 로직에 이벤트 ID 기록.
*   **효과:** 네트워크 오류로 인한 함수 재실행 시 중복 처리 방지.

### 3.3. Cold Start 최적화
*   **제안:** 주요 API 역할을 수행하는 함수에 `minInstances` 설정 적용.
*   **효과:** 사용자 요청 시 발생하는 초기 지연 시간(Latency) 최소화.

---

## 4. 사용자 경험(UX) 및 실시간성

### 4.1. 낙관적 업데이트 (Optimistic UI)
*   **제안:** '좋아요' 클릭 시 서버 응답 전 UI를 먼저 변경하고, 실패 시 롤백.
*   **효과:** 즉각적인 반응성 제공으로 사용자 만족도 향상.

### 4.2. 실시간 동기화 (onSnapshot) 범위 확장
*   **제안:** 채팅 외에 게시물 댓글 수, 실시간 인기 게시물 순위 등에 적용.
*   **효과:** 앱 전반에 생동감 부여.
