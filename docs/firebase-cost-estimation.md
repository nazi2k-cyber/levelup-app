## 2. 이용자 시나리오 정의

### 시나리오 A: 라이트 유저 (하루 1회 접속, 퀘스트만)
- 앱 실행 → 로그인 상태 확인 → 유저 문서 읽기 → 퀘스트 완료 → 저장
- 푸시 알림 수신 (레이드 3회 + 리마인더 1회)

### 시나리오 B: 일반 유저 (하루 2-3회 접속, 플래너+릴스 활용)
- 시나리오 A + 플래너 작성/수정 + 릴스 1개 포스팅 + 다른 유저 릴스 조회/좋아요
- 사진 업로드 1-2장 (플래너 or 릴스)

### 시나리오 C: 헤비 유저 (하루 5회+ 접속, 소셜 기능 적극 활용)
- 시나리오 B + 친구 프로필 조회 + 릴스 댓글/좋아요 다수 + 프로필 사진 변경

## 6. 비용 급증 위험 시나리오

### 6.1 릴스 피드 과다 조회
- 릴스 사진이 CDN 캐시 없이 직접 Storage에서 로드될 경우
- DAU 15,000 × 평균 20장 조회 × 1.5MB = **450GB/일 다운로드**
- **월 $1,600+ 가능**

### 6.2 getDocs() 전체 컬렉션 스캔
- `app.js`에서 `getDocs(collection(db, "users"))` 호출 시 모든 유저 문서를 읽음
- 유저 10만 명이면 1회 호출 = 100K 읽기
- 관리자가 아닌 일반 유저에서 호출되면 비용 폭증 위험

### 6.3 스트릭 경고 푸시 (sendStreakWarnings)
- 비활성 유저가 많을 경우 매일 전체 유저 스캔 + 개별 FCM 전송
- 유저 10만 명 → 10만 읽기 + 조건부 쓰기 매일

### 6.4 FCM 토큰 정리 (cleanupInactiveTokens)
- 매주 전체 유저 스캔 → 유저 수만큼 Firestore 읽기

## 7. 비용 최적화 권장사항

### 즉시 적용 가능

| 우선순위 | 항목 | 절감 효과 |
|----------|------|-----------|
| 🔴 높음 | **이미지 CDN 도입** (Firebase Hosting + Cache-Control 또는 CloudFlare) | Storage 다운로드 70~90% 절감 |
| 🔴 높음 | **이미지 리사이즈** (썸네일 생성 Cloud Function 추가) | 다운로드 용량 50~70% 절감 |
| 🟡 중간 | **Firestore 쿼리 최적화** - getDocs 대신 페이지네이션 적용 | 읽기 연산 30~50% 절감 |
| 🟡 중간 | **릴스 피드 캐싱** - 클라이언트 로컬 캐시 활용 확대 | 읽기/다운로드 절감 |
| 🟢 낮음 | **스케줄 함수 배치 처리** - 개별 send 대신 sendEach/sendEachForMulticast 활용 | Functions 실행 시간 절감 |

## 9. 결론

- **MAU 1만 이하**: 무료~월 2만원 수준으로 부담 없음
- **MAU 5만 기준**: 월 13~14만원, **Storage 다운로드가 전체 비용의 82%**
- **최우선 최적화 대상**: 이미지 CDN + 썸네일 리사이즈 도입 시 비용 60~80% 절감 가능
- **FCM, Analytics, Remote Config**: 무료이므로 비용 걱정 없음
- **Authentication**: MAU 5만까지 무료, Phone Auth 미사용으로 추가 비용 없음

> **핵심**: 릴스(Day1) 사진 다운로드가 최대 비용 요인. CDN 캐싱만 적용해도
> MAU 5만 기준 월 비용을 $105 → ~$30 수준으로 낮출 수 있음
