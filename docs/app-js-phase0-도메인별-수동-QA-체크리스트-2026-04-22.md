# app.js 도메인 모듈 분리 Phase 0 — 도메인별 수동 QA 체크리스트

- 작성일: 2026-04-22 (UTC)
- 범위: 로그인/퀘스트/플래너/광고/소셜/푸시
- 목적: 모듈 분리 작업 전/후 회귀 여부를 사람이 빠르게 비교 검증하기 위한 최소 체크 시나리오

## 공통 실행 규칙

1. 테스트 환경 고정
   - 브라우저: Chrome 최신 안정 버전(시크릿 모드 권장)
   - 계정: 테스트 계정 1개(일반), 관리자 계정 1개(선택)
2. 실행 순서
   - Baseline(현행) 1회 수행 후, 분리 단계별 동일 체크 재실행
3. 기록 방식
   - 각 항목 `PASS/FAIL`, 재현 절차, 콘솔 에러 유무를 함께 기록

---

## 1) 로그인 도메인

- [ ] 앱 첫 진입 시 로그인 상태에 맞는 화면 분기(로그인/메인)가 정상 동작한다.
- [ ] 이메일/소셜 로그인 완료 후 메인 화면까지 정상 진입한다.
- [ ] 로그아웃 후 세션이 종료되고 로그인 화면으로 복귀한다.
- [ ] 로그인 직후 사용자 포인트/프로필 카드가 정상 렌더링된다.

## 2) 퀘스트 도메인

- [ ] 오늘의 퀘스트 토글(`완료/미완료`)이 즉시 UI에 반영된다.
- [ ] DIY 퀘스트 생성/수정/삭제가 저장소와 UI에 동일 반영된다.
- [ ] 주간/월간 퀘스트 통계 UI(드롭다운/날짜 선택)가 정상 동작한다.
- [ ] 퀘스트 완료에 따른 보상(포인트/경험치)이 중복 없이 1회 반영된다.

## 3) 플래너 도메인

- [ ] 플래너 일정/할 일 추가·수정·삭제가 저장 후 새로고침 시 유지된다.
- [ ] 날짜 이동(일/주/월) 시 올바른 날짜 데이터가 렌더링된다.
- [ ] 전일 복사(할 일/스케줄)가 정상 작동한다.
- [ ] 플래너 공유(이미지/링크) 기능이 오류 없이 동작한다.

## 4) 광고 도메인

- [ ] 보상형 광고 진입/종료 이벤트가 정상 발생한다.
- [ ] 광고 시청 완료 시 보상이 정확히 1회 지급된다.
- [ ] 광고 실패/취소 시 보상이 지급되지 않는다.
- [ ] 광고 관련 콘솔 에러/네트워크 에러가 비정상적으로 증가하지 않는다.

## 5) 소셜 도메인

- [ ] 프로필 모달 열기/닫기 및 팔로우 토글 UI가 정상 동작한다.
- [ ] 소셜 링크(예: LinkedIn) 열기 시 새 창/탭 동작이 정상이다.
- [ ] 사용자 카드/타이틀 배지가 정상 렌더링된다.
- [ ] 소셜 피드/조회 로직에서 권한 에러가 새롭게 발생하지 않는다.

## 6) 푸시 도메인

- [ ] 푸시 권한 요청 팝업이 중복/무한 호출되지 않는다.
- [ ] 푸시 토글 ON/OFF 상태가 저장되고 재진입 후 유지된다.
- [ ] 권한 거부 상태에서 앱이 비정상 루프 없이 안전 동작한다.
- [ ] 인앱 알림 표시 함수 호출 시 UI가 정상 표시된다.

---

## 최종 판정 기준

- 치명 회귀(Critical): 로그인 불가, 데이터 저장 불가, 보상 중복 지급, 화면 진입 불가
- 주요 회귀(Major): 핵심 도메인 기능 1개 이상 실패
- 경미 회귀(Minor): UI/문구 문제, 우회 가능한 기능 불일치

> 권장: Phase별 배포 전/후 동일 체크리스트를 그대로 사용해 비교 가능성을 유지한다.
