# SPECS 개선 제안서

`SPECS.md` 기준으로 현재 아키텍처/이미지 파이프라인을 검토한 뒤, 우선순위 중심 개선안을 정리했습니다.

## 1) P0 (즉시 적용 권장)

### 1. 이미지 저장 경로 분리
- 현재 시간표(플래너) 이미지와 포스팅(릴스) 이미지가 `reels_photos/` 경로를 공유하고 있어 수명주기/권한 정책이 혼재될 여지가 있습니다.
- 제안:
  - 플래너 임시: `planner_photos/{uid}/{timestamp}.jpg`
  - 릴스 게시용: `reels_photos/{uid}/{postId}.jpg`
- 기대효과: 데이터 의미 명확화, 수동 운영/정리 시 실수 감소, 정책 분리 용이.

### 2. 관리자 권한 검증 방식 개선
- 현재 스펙상 관리자 식별을 단일 이메일 문자열에 의존하고 있습니다.
- 제안:
  - Firebase Auth Custom Claims (`admin: true`) 기반 권한 검증으로 전환
  - Firestore/Functions 모두 동일한 claim 기반 정책 사용
- 기대효과: 계정 변경/추가 시 운영 유연성 향상, 권한 오남용 리스크 축소.

### 3. 이미지 업로드 폴백(base64 직접 저장) 축소
- 업로드 실패 시 base64를 데이터로 유지하는 전략은 문서 크기 증가와 읽기 성능 저하를 유발할 수 있습니다.
- 제안:
  - 재시도(지수 백오프) 우선
  - 최종 실패 시에도 문서에는 실패 상태 + 로컬 재전송 큐만 저장
  - base64 영구 저장은 개발/디버그 플래그에서만 허용
- 기대효과: Firestore 문서 비대화 방지, 네트워크 불안정 상황에서 복구 가능성 향상.

## 2) P1 (단기 개선)

### 4. 업로드 API를 `uploadBytesResumable`로 전환
- 현재 30초 타임아웃 단일 업로드는 모바일 네트워크에서 실패율이 높을 수 있습니다.
- 제안:
  - resumable 업로드로 전환
  - 진행률 표시 + 재개 기능 추가
- 기대효과: 업로드 성공률 및 UX 개선.

### 5. 이미지 포맷 현대화(WebP 우선)
- 현재 JPEG 고정은 단순하지만 동일 화질 대비 용량 비효율이 있습니다.
- 제안:
  - 지원 브라우저에서 WebP 우선, 미지원 시 JPEG fallback
  - 프로필은 150x150 유지, 릴스는 최대변 600 기준 유지
- 기대효과: 저장/전송 비용 절감, 로딩 속도 개선.

### 6. Cloud Storage Lifecycle Rule 기반 정리
- 현재 만료 파일 정리를 스케줄 함수로 수행 중입니다.
- 제안:
  - 객체 메타데이터(`expireAt`)와 Lifecycle Rule 병행
  - 스케줄 함수는 예외 케이스 보정용으로 축소
- 기대효과: 운영 복잡도 감소, 정리 누락 가능성 축소.

## 3) P2 (중기 개선)

### 7. 클라이언트 이미지 처리 워커 분리
- 모든 리사이즈/압축이 메인 스레드에서 이뤄지면 저사양 단말에서 UI 끊김이 발생할 수 있습니다.
- 제안:
  - Web Worker + OffscreenCanvas(가능한 플랫폼) 적용
- 기대효과: 체감 성능 개선, 이미지 처리 중 UX 안정화.

### 8. 데이터 모델 분리(핫/콜드 데이터)
- `users/{uid}` 단일 문서에 많은 상태가 모이면 읽기 비용과 충돌 가능성이 증가합니다.
- 제안:
  - `users/{uid}`: 프로필/핵심 상태
  - `users/{uid}/daily/{date}`: 일일 상태
  - `reels/{postId}`: 피드 전용
- 기대효과: 동시 업데이트 충돌 감소, 쿼리 단순화, 확장성 향상.

### 9. 관측성(Observability) 강화
- 푸시/정리 작업은 있으나 실패율, 지연, 재시도 지표가 구조적으로 드러나지 않습니다.
- 제안:
  - Cloud Logging 구조화 로그 + 에러율 알림(SLO 기반)
  - 주요 함수별 대시보드(성공률, 평균 처리시간, 에러코드 TOP N)
- 기대효과: 장애 조기 감지 및 원인 파악 속도 향상.

## 4) 바로 실행 가능한 2주 액션 플랜

1. **1주차**
   - 관리자 권한을 Custom Claims로 전환
   - 업로드 재시도/백오프 로직 추가
   - 경로 분리 스키마 설계 및 신규 업로드부터 적용

2. **2주차**
   - `uploadBytesResumable` 전환 + 진행률 UI
   - WebP 우선 인코딩 도입
   - Lifecycle Rule PoC 적용

## 5) 성공 지표(KPI) 제안

- 이미지 업로드 실패율: **현재 대비 30%↓**
- 평균 이미지 용량: **현재 대비 25%↓**
- 업로드 완료까지 평균 시간: **현재 대비 20%↓**
- 만료 이미지 정리 누락률: **0%에 수렴**
